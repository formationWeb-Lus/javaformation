Semaine 02 – Activité d’Apprentissage : Menus Responsives

Vue d’ensemble

Le but de cette activité est de construire un menu responsive en utilisant JavaScript. Le menu s’adaptera à différentes tailles d’écran et réagira également lorsqu’un utilisateur cliquera sur le bouton « hamburger » (≡). En conception responsive, notre objectif est d’offrir aux utilisateurs une expérience fluide et intuitive.

Résultat d’apprentissage associé

Développer des pages web responsives qui respectent les bonnes pratiques et utilisent un HTML et CSS valides.

Préparation

Les menus hamburger ou les structures équivalentes soutiennent la conception responsive. Ils permettent d’alléger la mise en page en économisant de l’espace sur la page. Ce symbole est largement reconnu par les utilisateurs, même s’ils ne savent pas toujours qu’il s’appelle « menu hamburger ».

Instructions de l’activité

  1. Dans votre propre compte CodePen, ouvrez ce CodePen ☼ Menu Responsive – Départ
  2. CSS : Déplacez le bouton du menu hamburger dans le coin supérieur droit de l’écran en ajoutant les déclarations CSS appropriées aux endroits indiqués (______).
  3. JavaScript : Complétez le code JavaScript en remplissant les trois (3) zones vides (______) à l’intérieur de l’écouteur d’événement du bouton de menu.
    Cet exemple écoute un événement déclenché par l’utilisateur et alterne la classe donnée pour l’élément de navigation. Le nom de la classe existe déjà dans le CSS et est prêt à être appliqué.
  4. CSS : Changez l’icône de fermeture du bouton hamburger (❎) pour une autre icône de votre choix.
    5. Que fait la syntaxe :: dans un sélecteur CSS ?

    Cette syntaxe permet de sélectionner un pseudo-élément, ce qui permet d’ajouter du contenu à un élément existant sans modifier le HTML.

  5. CSS : Repérage (Wayfinding) – Ajoutez une règle pour la classe active afin d’indiquer à l’utilisateur quelle page est actuellement ouverte, en modifiant visuellement son apparence.
    Exemple : 
    <a href="#" title="À propos" class="active">À propos</a>

    « Faciliter la navigation sur un site ou une application aide tout le monde à trouver rapidement ce dont il a besoin. Un repérage clair est particulièrement utile pour les personnes ayant des difficultés visuelles, motrices ou cognitives, qui peuvent autrement avoir du mal à comprendre où elles se trouvent et comment atteindre leur destination. » – Université Harvard – Accessibilité numérique

  6. N’oubliez pas de Sauvegarder vos modifications sur votre CodePen.
