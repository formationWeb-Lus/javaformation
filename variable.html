Semaine 01 : Activité – Variables en JavaScript

Vue d’ensemble

Les variables sont utilisées pour stocker des valeurs. Ces valeurs peuvent être primitives comme un nombre, une chaîne (string), un booléen ou null, ou bien des objets comme une fonction, une classe ou un objet littéral. Elles peuvent être utilisées comme entrée dans des expressions, lors de traitements ou pour afficher des résultats.

Pour mieux comprendre le fonctionnement de JavaScript, ce cours couvre les concepts de base et les structures de contrôle, y compris les variables. Ces structures sont communes à tous les langages de programmation.

Préparation

Répondez aux questions suivantes en utilisant ces ressources ou vos propres recherches :

  • Stocker les informations — Variables – MDN
  • Types et structures de données – MDN
  1. Quelle est la différence entre const et let ?
    Vérifiez votre compréhension
    • const : Déclare une variable constante qui ne peut pas être réassignée. 
      const currentYear = new Date().getFullYear();
    • let : Déclare une variable réassignable. 
      let isLoggedIn = false;
  2. Définissez le type de données string.
    Vérifiez votre compréhension

    Une chaîne de caractères est une séquence de symboles (lettres, chiffres, espaces, etc.) considérée comme une seule unité.

  3. Quel est le type de données de chacun des éléments suivants ?
    • "101"
    • 101
    • true
    • null
    • undefined
  4. Quelles guillemets utiliser pour entourer les chaînes : simples ('), doubles ("), ou backticks (`) ?
    Vérifiez votre compréhension

    En JavaScript, vous pouvez utiliser les trois, mais les backticks sont utiles pour les chaînes multilignes et les expressions dynamiques.

  5. Qu’est-ce qu’un template literal et pourquoi l’utiliser ?
    Vérifiez votre compréhension

    Ce sont des chaînes entourées de backticks qui permettent d’insérer des variables et expressions facilement.

  6. Qu’est-ce qu’un commentaire de code en JavaScript ?
    Vérifiez votre compréhension

    Les commentaires servent à annoter le code, par exemple :

    // Ceci est un commentaire
/* Ceci est un commentaire multi-ligne */
